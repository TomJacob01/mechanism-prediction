# Data

`mech_uspto` trains on the **mech-USPTO-31k** multi-step reaction dataset.
The data is **not bundled** with this repository — you must obtain it
separately and point the training scripts at it.

## Obtaining the dataset

The dataset is published under **CC BY 4.0** and hosted on figshare:

- **Figshare record:** <https://figshare.com/articles/dataset/mech-USPTO-31k_reaction_dataset/24797220>
- **Companion paper (Nature Scientific Data, 2024):**
  Joung, Mobasher et al., *"A large-scale reaction dataset of mechanistic
  pathways of organic reactions"*,
  <https://www.nature.com/articles/s41597-024-03709-y>
  (open access mirror: <https://pmc.ncbi.nlm.nih.gov/articles/PMC11316731/>)

> ⚠️ **Format caveat.** The figshare archive may not ship as one JSON file
> per reaction — the original release format is a single tabular / SMILES file
> generated by MechFinder. The parser in this repo
> ([`MechUSPTOParser`](src/mech_uspto/data/parser.py)) expects the
> per-reaction JSON layout shown below. **Verify the on-disk format before
> training**: if you get a different layout (e.g. a single `.txt` /
> `.csv` / `.tsv` of SMILES + arrow strings), you'll need a small
> conversion script that emits one JSON per reaction matching the schema in
> the next section. Run `python scripts/sanity_check.py --data-dir <path>`
> immediately after extraction — it will fail loudly with a schema error if
> the format doesn't match, before you spend GPU hours.

### Manual download

1. Open the figshare page above and download the dataset archive.
2. Extract it to a directory of your choice, e.g. `~/data/mech-USPTO-31k/`.
3. If the contents are a single tabular file rather than per-reaction JSONs,
   run a conversion step (see "Format caveat" above).
4. Run the sanity check before training (see bottom of this file).

### Programmatic download (optional)

You can script the download with `wget`/`curl` once you have the direct
file URL from the figshare page (figshare assigns versioned URLs of the form
`https://ndownloader.figshare.com/files/<file_id>`). The exact `file_id`
depends on the archive version — pick it off the figshare page rather than
hard-coding it here.

## Pointing the scripts at the data

Either pass `--data-dir` on the command line, or set the environment variable
`MECH_USPTO_DATA`:

```bash
export MECH_USPTO_DATA=~/data/mech-USPTO-31k
python scripts/sanity_check.py
python scripts/train.py --task-mode stepwise --num-epochs 100
```

If neither is provided, the scripts fall back to `/cluster/data/mech-USPTO-31k`.

## Expected directory layout

```
mech-USPTO-31k/
├── rxn_00001.json
├── rxn_00002.json
├── ...
└── rxn_31xxx.json
```

One JSON file per reaction.

## JSON schema

Each file describes one multi-step reaction:

```json
{
    "rxn_id": "rxn_0001",
    "steps": [
        {
            "id": 0,
            "reactants": "SMILES",
            "products": "SMILES",
            "reactants_mapped": "atom-mapped SMILES",
            "products_mapped": "atom-mapped SMILES",
            "mechanism": "label or arrow code"
        }
    ],
    "overall_reactants": "SMILES",
    "overall_products": "SMILES",
    "metadata": {"...": "..."}
}
```

Required keys: `rxn_id`, `steps` (non-empty list), and within each step
`reactants_mapped` and `products_mapped`. Atom map numbers must be consistent
between reactants and products of the same step (this is what lets the
parser align atoms and compute the bond-order Δ matrix).

## Verifying a data drop

Before launching a long training run, run the pre-flight check:

```bash
python scripts/sanity_check.py --data-dir <path>
```

It validates the directory layout, schema, parsing, featurization,
Δ statistics (must lie in `{-1,0,1}` for stepwise / `{-2,-1,0,1,2}` for
end-to-end), spectator ratio, and runs one mini training step end-to-end.
See the script's `--help` for flags.

## Featurization summary

- Node features: 25-dim (one-hot element + degree + formal charge + hybridisation
  + aromaticity + H-count). See `mech_uspto.constants`.
- Edge features: 6-dim (bond type one-hot + conjugation + ring membership).
- Targets: bond-order Δ adjacency matrices, padded to `(B, N_max, N_max)` and
  shifted to non-negative class indices inside the training engine.
